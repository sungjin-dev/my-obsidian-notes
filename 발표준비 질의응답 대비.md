
 데이터가 이동할 때의 전체 과정

1. 모달창을 켜고 사용자가 텍스트를 입력하고 save 버튼을 누르면 save 담당 자바스크립트함수로 넘어감

2. 해당 함수에서 getElementById(ID값)으로 입력된 데이터 받아 데이터 수집 (Data Collection)하고 새로운 변수명으로 상수화한다. 

3. 페이로드(Payload)를 만든다. JSON 객체로 구조화한다. 데이터 캡슐화(Encapsulation)
   const dataToSend = { 
     resolution : newResolution, 
     progress : newProgress, 
     errorid : currentEnum 
    }
    
4. fetch()에 보낼 url주소를 적는다. ex) 백엔드 서버. 메서드는 GET, POST 방식으로 적어두고 header에는 어떤 종류의 데이터인지 적는다. 'application/json'

5. body부분에 JSON.stringify()  (json모듈함 공구) 을 사용해서 아까 만들어준 json 객체를 집어 넣어 인터넷 회선을 타고 이동할 수 있는 '텍스트(문자열)'로 포장한다 
    이 텍스트들이 0과 1의 전기 신호로 바뀌어 회선을 통과합니다.

6. 백엔드 서버에서 회선을 통해 들어온 신호를 다시 '텍스트'로 복구합니다. 잘 받았으면 프론트엔드에 그 텍스트를 다시 객체로 변환해서 응답을 보낸다. 

// 서버가 보낸 영수증 덩어리(응답) 
{ 
    "status": 200, // 상태 코드 
    "headers": { // 헤더 정보 
    "Content-Type": "application/json" 
    }, 
    "body": "{\"success\":true,\"savedId\":123}" // 이 부분을 .json()으로 뜯는 것! 
    }

    then 부분에서 영수증에 response.ok을 통해 서버로부터 받은 영수증을 보고 에러를 탐지

내가 자동으로 JSON.parse() 해서 객체로 만들어줄게!"라고 미리 다 처리를 해버립니다.
